# SEO記事生成ルール｜神奈川まごころ整理センター

## 対象読者プロフィール

- **年齢・性別**：40代〜60代女性が中心（男性も一定数いる）
- **状況**：実家の片付け・引越し・親の遺品整理・断捨離など「今すぐ解決したい悩み」を抱えている
- **ITリテラシー**：スマートフォン中心。複雑な情報より「結論→理由→行動」の流れを好む
- **悩み**：「料金が不安」「信頼できる業者かどうか不安」「自分で運び出せない」
- **求めるもの**：安心感・スピード・丁寧な対応・明確な料金

---

## 文体・トーン

- **語尾**：ですます調を徹底（「〜です」「〜ます」「〜ください」）
- **漢字率**：やや低め。難しい漢字はひらがなに開く（例：「および」→「および」でもOKだが「又」→「また」に統一）
- **句読点**：1文40字以内を目安。長くなったら分割する
- **感嘆符・強調**：過度に使わない。重要箇所は `<strong>` タグで強調
- **AI臭を消すポイント**（下記参照）

---

## AI臭を消すルール

以下の表現は必ず避ける。これらが入っていたら書き直す。

| 避ける表現 | 代わりに使う |
|------------|-------------|
| 〜してみましょう | 〜しましょう／〜がおすすめです |
| 〜することが大切です | 〜が重要です／〜を心がけてください |
| 〜と言えるでしょう | 〜です／〜といえます |
| 非常に重要 | 大切な／重要な |
| 〜ということが挙げられます | 〜があります |
| まず初めに | まず |
| それでは | （削除） |
| いかがでしたか | （削除） |
| 以上が〜です | （削除、記事を締めるだけでOK） |
| ChatGPT的な箇条書き乱発 | 適切に段落文章で説明する |

**自然な語り口のコツ：**
- 読者に直接語りかける（「〜でお困りではありませんか？」）
- 実例・具体的な数字を使う（「最短30分」「平均2万円前後」）
- 現場感のある言葉を使う（「実際によくあるご相談です」「よくいただくご質問です」）

## 記事構成テンプレート

```
【タイトル】
[主要キーワード]！[補足情報]
例：大和市で不用品回収を即日対応！費用目安と業者選びのポイント

【メタディスクリプション】
100〜130字。キーワード・具体的な数字・CTAを含める。
例：大和市で不用品回収を即日対応！最短30分で到着、平均2万円前後。
    買取で費用ゼロのケースも。追加料金なし・許可取得済みの業者選びポイントを解説。

【記事構成】

1. リード文（200〜300字）
   - 読者の悩み・状況に共感する一文から始める
   - 「この記事でわかること」を簡潔に示す
   - キーワードを自然に含める

2. h2：[状況・背景の説明]（300〜400字）
   - 読者がなぜその問題に直面しているかを説明
   - 自治体との比較や一般的な状況を示す

3. h2：[料金・費用の目安]（300〜400字）
   - 必ず料金表（price-table）を入れる
   - 「※あくまで目安」の注釈を必ず入れる
   - 「無料見積もり」に誘導する

4. h2：[業者・サービスの特徴]（400〜500字）
   - 会社の強みを自然に織り込む
   - 「早い・丁寧・紹介」「追加料金なし」「即日対応」など
   - info-box を使って強調

5. h2：[比較表または注意点]（300〜400字）
   - 「自治体 vs 業者」「良い業者 vs 悪徳業者」など
   - 必ず比較表（price-table）を入れる

6. h2：[ポイント・まとめ]（300〜400字）
   - 3〜4項目を h3 で区切って説明
   - 読者が「次に何をすればいいか」がわかるように書く

7. article-cta（CTAブロック）
   - 電話番号 0120-437-599
   - LINE https://lin.ee/Gf8Or5r
   - 「しつこい勧誘なし」の一言

8. 関連コラム（related-articles）
   - 既存記事から3本選んでリンク
```

## SEOルール

### タイトルタグ
- 30〜35字以内
- 狙うキーワードを先頭に置く
- `｜神奈川まごころ整理センター` を末尾に必ずつける

### 見出し（h1〜h3）
- h1：記事タイトルと同じ（1本のみ）
- h2：主要セクションの見出し。キーワードを自然に含める
- h3：h2 の補足説明。「ポイント①」「コツ②」のような連番が読みやすい

### 内部リンク
- **必須**：関連ブログ記事3本（related-articles セクション）
- **必須**：エリア記事の場合は `/area/xxx.html` へのリンク
- **推奨**：記事本文中にも1〜2本の内部リンクを自然に挿入

### キーワード配置
- タイトル：必ず入れる
- h1：タイトルと同じで自動的にOK
- メタディスクリプション：必ず入れる
- リード文（最初の100字以内）：必ず入れる
- 本文中：3〜5回程度（詰め込みすぎない）

### 構造化データ（JSON-LD）
- `<head>` 内に Article スキーマを必ず入れる
- `datePublished` は記事作成日（YYYY-MM-DD形式）
