# SEO記事生成ルール｜神奈川まごころ整理センター

## 対象読者プロフィール

- **年齢・性別**：40代〜60代女性が中心（男性も一定数いる）
- **状況**：実家の片付け・引越し・親の遺品整理・断捨離など「今すぐ解決したい悩み」を抱えている
- **ITリテラシー**：スマートフォン中心。複雑な情報より「結論→理由→行動」の流れを好む
- **悩み**：「料金が不安」「信頼できる業者かどうか不安」「自分で運び出せない」
- **求めるもの**：安心感・スピード・丁寧な対応・明確な料金

---

## 文体・トーン

- **語尾**：ですます調を徹底（「〜です」「〜ます」「〜ください」）
- **漢字率**：やや低め。難しい漢字はひらがなに開く（例：「および」→「および」でもOKだが「又」→「また」に統一）
- **句読点**：1文40字以内を目安。長くなったら分割する
- **感嘆符・強調**：過度に使わない。重要箇所は `<strong>` タグで強調
- **AI臭を消すポイント**（下記参照）

## AI臭を消すルール

### ❌ 絶対に使わない表現リスト

以下の表現が1つでも入っていたら、その段落ごと書き直す。

| 避ける表現 | 代わりに使う |
|------------|-------------|
| 〜してみましょう | 〜しましょう／〜をおすすめします |
| 〜することが大切です | 〜が肝心です／〜を忘れずに |
| 〜と言えるでしょう | 〜です／〜だと思います |
| 非常に重要／非常に大切 | 特に大事な／見落とせない |
| 〜ということが挙げられます | 〜があります／たとえば〜です |
| まず初めに | まず |
| それでは〜 / では〜を見ていきましょう | （削除してそのまま本題へ） |
| いかがでしたか | （削除。締めはCTAへ自然につなぐ） |
| 以上が〜です／〜についてご紹介しました | （削除） |
| 〜する際には注意が必要です | 〜には気をつけてください／〜は要注意です |
| 〜のポイントを押さえておきましょう | 〜を確認しておくと安心です |
| 様々な〜 / さまざまな〜 | 複数の〜／いくつかの〜（または具体例を列挙） |
| 〜することができます | 〜できます |
| ぜひ〜してみてください | 〜してみてください／〜をお試しください |
| 〜をしっかりと確認する | 〜を確認する |
| 〜という点が重要です | 〜が重要です |
| 総じて〜 / 全体的に〜 | （削除か具体的な言葉に置換） |

### ❌ 構造のAI臭パターン（これも書き直し対象）

- **箇条書きの乱発**：3つ以上の h3 が連続して全部「①②③」形式になっている
- **全段落が同じ長さ**：どの段落も3〜4文でぴったり揃っている
- **接続詞の単調な繰り返し**：「また〜」「さらに〜」「そして〜」が連続する
- **見出しの平行構造すぎる**：「〜の方法」「〜のポイント」「〜の注意点」が規則正しく並ぶ
- **結論が毎回同じ型**：各セクションの締めが「〜です。ぜひ参考にしてください。」のパターン

## 人間らしい文章を書く技術

### ① 文の長さをわざとバラバラにする

人間の文章は短文と長文が混在する。AIは均一になりがち。

```
❌ AIっぽい例：
大和市での不用品回収は、業者によって対応速度が大きく異なります。
当センターでは最短30分でお伺いできます。費用は平均2万円前後です。

✅ 人間らしい例：
大和市の場合、当センターの事務所が福田にあるので、最短30分で伺えます。
これが他社との一番の違いです。
費用については「いくらかかるか正直に教えてほしい」というお声をよくいただくので、
まず目安から正直にお伝えします。平均で2万円前後、というのが実感値です。
```

### ② 「実は」「正直なところ」「よくあるのが」を使う

現場で働く人が話すような言葉を意識して使う。

- 「実は、この質問が一番多いんです」
- 「正直なところ、自治体回収で間に合うケースもあります」
- 「よくあるのが、捨てるつもりだったものが買取になるパターンです」
- 「意外に思われるかもしれませんが〜」
- 「ここだけの話、〜」（使いすぎ注意）

### ③ 読者の心の声を代弁する

「〜でお困りではないですか？」より一歩踏み込んで、読者が頭の中で思っていることを先に言う。

```
❌ 平凡な例：
不用品回収の費用が気になる方も多いと思います。

✅ 刺さる例：
「見積もりを取ったら思ったより高くて断れなかった」——そんな経験、ありませんか。
```

### ④ 具体的な固有名詞・数字を使う

抽象的な表現を避け、固有の情報を入れる。これはAI検出ツールを通過する上でも有効。

- ❌「短時間で対応できます」→ ✅「最短30分で到着します」
- ❌「多くの実績があります」→ ✅「累計1,200件以上の実績があります」
- ❌「費用がかからないこともあります」→ ✅「買取金額が回収費用を上回り、実質ゼロ円になったケースが実際にあります」

### ⑤ 完璧すぎない文章を書く

人間の文章には多少の「ゆれ」がある。

- 同じことを2通りの言い方で言い直す箇所があってもいい
- 「〜ですが、」で始まる逆説の文を混ぜる
- 体言止めを時々使う（「これが大事なポイントです」→「これが大事。」）
- 読点の位置を均一にしない

### ⑥ 段落の冒頭をバラバラにする

AIは段落を「〜は〜です。」「〜は〜ます。」と同じパターンで始めがち。

冒頭を意識的に変化させる：
- 「実は〜」
- 「たとえば、〜」
- 「よく聞かれるのが〜」
- 「〜というと、〜」
- 「意外かもしれませんが、〜」
- 「一番多い理由は〜」
- 「気をつけたいのが〜」
- 短い問いかけ文「費用はどれくらいかかるのでしょうか。」

### ⑦ 最後はCTAへ自然につなぐ（まとめ文を書かない）

AIは記事末に「以上、〜についてご紹介しました。ぜひ参考にしてください。」と書きがち。
そのまとめ文は不要。最後のセクションの流れから自然にCTAへつなぐ。

```
❌ AI的な締め：
以上、大和市での不用品回収についてご紹介しました。ぜひ参考にしてください。

✅ 自然な締め：
まずは「どれくらいかかるか」だけでも教えてほしい、という段階でも大丈夫です。
お電話かLINEで、気軽にご連絡ください。
```

## 記事構成テンプレート

```
【タイトル】
[主要キーワード]！[補足情報]
例：大和市で不用品回収を即日対応！費用目安と業者選びのポイント

【メタディスクリプション】
100〜130字。キーワード・具体的な数字・CTAを含める。
例：大和市で不用品回収を即日対応！最短30分で到着、平均2万円前後。
    買取で費用ゼロのケースも。追加料金なし・許可取得済みの業者選びポイントを解説。

【記事構成】

1. リード文（200〜300字）
   - 読者の悩み・状況に共感する一文から始める
   - 「この記事でわかること」を簡潔に示す
   - キーワードを自然に含める

2. h2：[状況・背景の説明]（300〜400字）
   - 読者がなぜその問題に直面しているかを説明
   - 自治体との比較や一般的な状況を示す

3. h2：[料金・費用の目安]（300〜400字）
   - 必ず料金表（price-table）を入れる
   - 「※あくまで目安」の注釈を必ず入れる
   - 「無料見積もり」に誘導する

4. h2：[業者・サービスの特徴]（400〜500字）
   - 会社の強みを自然に織り込む
   - 「早い・丁寧・紹介」「追加料金なし」「即日対応」など
   - info-box を使って強調

5. h2：[比較表または注意点]（300〜400字）
   - 「自治体 vs 業者」「良い業者 vs 悪徳業者」など
   - 必ず比較表（price-table）を入れる

6. h2：[ポイント・まとめ]（300〜400字）
   - 3〜4項目を h3 で区切って説明
   - 読者が「次に何をすればいいか」がわかるように書く

7. article-cta（CTAブロック）
   - 電話番号 0120-437-599
   - LINE https://lin.ee/Gf8Or5r
   - 「しつこい勧誘なし」の一言

8. 関連コラム（related-articles）
   - 既存記事から3本選んでリンク
```

## SEOルール

### タイトルタグ
- 30〜35字以内
- 狙うキーワードを先頭に置く
- `｜神奈川まごころ整理センター` を末尾に必ずつける

### 見出し（h1〜h3）
- h1：記事タイトルと同じ（1本のみ）
- h2：主要セクションの見出し。キーワードを自然に含める
- h3：h2 の補足説明。「ポイント①」「コツ②」のような連番が読みやすい

### 内部リンク
- **必須**：関連ブログ記事3本（related-articles セクション）
- **必須**：エリア記事の場合は `/area/xxx.html` へのリンク
- **推奨**：記事本文中にも1〜2本の内部リンクを自然に挿入

### キーワード配置
- タイトル：必ず入れる
- h1：タイトルと同じで自動的にOK
- メタディスクリプション：必ず入れる
- リード文（最初の100字以内）：必ず入れる
- 本文中：3〜5回程度（詰め込みすぎない）

### 構造化データ（JSON-LD）
- `<head>` 内に Article スキーマを必ず入れる
- `datePublished` は記事作成日（YYYY-MM-DD形式）
