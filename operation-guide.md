# SEOブログ運用ガイド｜神奈川まごころ整理センター

## 目的
ブログ記事を継続的に追加し、「大和市 不用品回収」「神奈川 遺品整理」などのロングテールキーワードで検索流入を増やす。月1〜2本の新記事を安定公開し、6か月後に月間検索流入50件以上を目指す。

---

## 運用ペース目安

| フェーズ | 期間 | 投稿ペース | 目標記事数 |
|----------|------|------------|-----------|
| 立ち上げ | 1〜2か月目 | 週1本 | 8〜10本 |
| 安定期 | 3〜6か月目 | 隔週1本 | 20本前後 |
| 維持期 | 7か月目以降 | 月1本 | 随時追加 |

**最低ライン：月2本**。それ以下になると検索エンジンに「更新頻度が低いサイト」と判断されやすい。

## 毎週の作業手順（所要時間：15〜30分）

### STEP 1：キーワードを決める（2分）
1. `keyword-queue.md` を開く
2. 未着手（[済]がついていない）キーワードの中から上から順に1つ選ぶ
3. そのキーワードをメモしておく

### STEP 2：Claude Codeに記事生成を依頼（10〜20分）
1. `weekly-command.md` を開く
2. テンプレートをコピーし、`【キーワード】` 部分を選んだキーワードに書き換える
3. Claude Codeに貼り付けて実行
4. 生成された記事を確認（内容・事実確認・リンク切れチェック）

### STEP 3：公開確認（3〜5分）
1. git push が完了したことを確認
2. VercelのデプロイURLで実際のページを確認
3. スマートフォンで表示崩れがないかチェック

### STEP 4：キーワード管理（1分）
1. `keyword-queue.md` の使用したキーワードに `[済]` をつける
2. 公開日付も記録する（例：`[済 2026-04-14]`）

## 記事1本あたりのSEO品質チェック

公開前に以下を必ず確認する。

- [ ] タイトルに狙うキーワードが含まれている
- [ ] メタディスクリプションが120字前後でまとまっている
- [ ] h2・h3 の見出しが論理的に並んでいる
- [ ] 電話番号 0120-437-599 と LINE リンクが記事内とCTAに入っている
- [ ] 関連記事リンクが3本以上ある
- [ ] エリアページ `/area/xxx.html` への内部リンクがある（エリア記事の場合）
- [ ] 比較表または料金表が1つ以上入っている
- [ ] 「ですます調」で統一されている
- [ ] 文字数が2,000字以上ある
- [ ] `blog/index.html` に新記事カードが追加されている

## ファイル名のルール

```
[エリア]-[サービス略称]-[特徴].html

例：
yamato-fuyo-sokuji.html    → 大和市 不用品回収 即日
yokohama-ihin-cost.html    → 横浜市 遺品整理 費用
kanagawa-fuyo-single.html  → 神奈川 不用品回収 単品
```

- 日本語・スペース・大文字は使わない
- すべて半角英数字とハイフンのみ
- 長くなりすぎない（4単語以内）

---

## 注意点・やってはいけないこと

### コンテンツ面
- **同じキーワードで2本作らない**：`keyword-queue.md` を必ず確認する
- **事実を盛らない**：「最安値保証」「業界No.1」など根拠のない表現は書かない
- **料金を断言しすぎない**：「〜円〜」の幅表記を使い、「※あくまで目安」を添える
- **競合他社名を出さない**：「A社より安い」等の記述は避ける

### 技術面
- `git push` 前に必ずローカルで HTML の見た目を確認する
- `blog/index.html` への追加を忘れない（これを忘れると記事がブログ一覧に出ない）
- 画像を使う場合は必ず最適化（WebP推奨、100KB以下）してから追加する
- 既存の `style.css` を書き換えない

## 月次レビュー（月1回・15分）

毎月末に以下を確認して方針を調整する。

1. **Google Search Console**：表示回数・クリック数が増えているキーワードを確認
2. **上位表示された記事**：さらに内部リンクを増やして強化
3. **検索されているのに記事がないキーワード**：次月のキーワードリストに追加
4. **直帰率が高い記事**：CTAの位置や文言を改善

## 困ったときの対処

| 問題 | 対処法 |
|------|--------|
| git push でエラー | `git pull origin main` してから再度 push |
| 記事がブログ一覧に出ない | `blog/index.html` へのカード追加を確認 |
| Vercelでデプロイが失敗 | HTMLの構文エラーを確認（タグの閉じ忘れなど） |
| キーワードが思いつかない | `keyword-queue.md` の未着手分から選ぶ |
| 記事の品質に自信がない | `article-prompt.md` の品質チェックリストを再確認 |
